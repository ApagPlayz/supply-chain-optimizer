**Fetch graph metrics when switching to Network Risk view:**
```tsx
useEffect(() => {
  if (mapView !== 'network-risk') return;
  if (graphMetrics) return;  // already loaded — don't re-fetch
  graphAPI.metrics()
    .then((res) => {
      const metrics: GraphMetricsResponse = res.data;
      setGraphMetrics(metrics);
      // Derive single-source distributors: distributors with betweenness > 0
      // that are sole source of any component.
      // For simplicity, any distributor with betweenness > 0.5 (normalized)
      // is treated as single-source-relevant — a rough proxy since
      // single_source_component_ids is not exposed per-distributor in /graph/metrics.
      // The authoritative single-source list uses component-level join below.
      // For the side panel, we derive the list by intersecting distributor IDs
      // with the betweenness dict (high-betweenness = likely sole source in this bipartite graph).
      // The exact single-source list comes from a separate API call to /graph/metrics
      // which exposes single_source_count but not per-component breakdown.
      // Use betweenness > 0.6 as the threshold for the red halo:
      const soloSourceIds = new Set<number>(
        Object.entries(metrics.betweenness)
          .filter(([, b]) => b > 0.6)
          .map(([id]) => parseInt(id))
      );
      setSingleSourceDistributors(soloSourceIds);
      // Build side panel entries from distributors list (already in state from existing fetch)
      // Cross-reference with soloSourceIds to get distributor names
      // singleSourceComponents list: the API doesn't expose per-component→distributor mapping
      // so create entries for each high-betweenness distributor indicating it's a network hub
      const panelEntries = (distributors as DistributorPin[])
        .filter((d) => soloSourceIds.has(d.id))
        .map((d) => ({
          mpn: `High-betweenness hub`,
          manufacturer: d.country,
          distributor_name: d.name,
          distributor_id: d.id,
        }));
      setSingleSourceComponents(panelEntries);
    })
    .catch(() => {
      // Silent fail — Network Risk view shows unsized markers on API error
    });
}, [mapView]);
```

## Trust Boundaries

|----------|-------------|
| Browser → /graph/metrics | Reads aggregate graph analytics — no user data |
| Browser → /benchmark/cascade-heatmap | Reads per-distributor collapse probabilities — no user data |
| API response → maplibre layer | GeoJSON feature properties injected into map layer paint expressions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04-01 | Information Disclosure | /graph/metrics exposes distributor betweenness + k-core data | accept | All data is aggregate topology metrics. No prices, no user PII, no credentials. Mirror of graph.py T-02-04 accept disposition. |
| T-04-04-02 | Tampering | cascadeHeatmapData injected into maplibre GeoJSON features as `properties.weight` | mitigate | `weight` is a numeric float from the API response used only in `heatmap-weight` paint expression (never rendered as DOM text). Paint expressions are evaluated by maplibre internally — not eval'd as JavaScript. No injection vector. |
| T-04-04-03 | Information Disclosure | distributor names rendered in side panel tooltip aria-label | accept | Distributor names are public business names (DigiKey, Mouser, etc.) — no sensitive data. |
| T-04-04-04 | Denial of Service | cascadeHeatmapData array renders as GeoJSON source | mitigate | Bounded by n_distributors (92 max). Each feature is a Point with 2 properties. No risk of browser OOM. |
| T-04-04-05 | Spoofing | mapRef.current?.flyTo() called from side panel row click | mitigate | flyTo only runs when `dist` is found in the local `distributors` array (already-loaded, server-provided data). No URL is constructed from user input. |
