Current BUGGY code on line 514 (inside the Fiedler-degradation LineChart's <Tooltip>):
```tsx
formatter={(value: number) => [value.toFixed(4), 'λ₂']}
```

Why it fails:
- recharts' Tooltip formatter is typed `Formatter<ValueType, NameType>` where
  `ValueType = number | string | Array<number | string>`.
- The callback signature recharts exposes passes `value: ValueType | undefined`.
- Annotating the parameter as `: number` narrows the input incompatibly — tsc raises
  TS2322 "Type '(value: number) => [string, string]' is not assignable to type
  'Formatter<ValueType, NameType>'".

Locked fix (per 04-VERIFICATION.md "missing" field):
```tsx
formatter={(value) => [typeof value === 'number' ? value.toFixed(4) : '—', 'λ₂']}
```

This:
- Removes the too-narrow `: number` annotation so TS infers the recharts-compatible type.
- Adds a `typeof value === 'number'` guard so `.toFixed(4)` is only called on numbers.
- Falls back to U+2014 em dash `—` (NOT two hyphens `--`) for non-number values.
- Keeps the second tuple element `'λ₂'` (U+03BB lambda + U+2082 subscript 2) unchanged.
