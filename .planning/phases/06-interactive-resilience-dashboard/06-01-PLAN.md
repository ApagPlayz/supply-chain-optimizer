<objective>
Build three scenario simulation endpoints that compute cost/ETA/risk deltas when distributors fail, geopolitical risk spikes, or delivery targets tighten. This plan creates the backend API layer that ResiliencePage will call to produce "what if" results. All endpoints return baseline + scenario values, P10/P50/P90 fulfillment rates, and affected BOM lists.

Purpose: RESIL-01, RESIL-02, RESIL-03 require fast, interactive scenario exploration. These endpoints orchestrate existing Phase 2-5 simulation logic (graph cascade, live feed overrides, re-optimization) and cache results (1h TTL) to meet <2s response time.

Output: Three new POST endpoints at /resilience/{scenario-type}, a ScenarioCache ORM table with cache_key/result/expires_at, Alembic migration, and test scaffold with RED assertions.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → /resilience/* endpoints | Untrusted BOM component IDs and distributor IDs from request body |
| /resilience/* → Graph/Live Feeds | Request parameters that override live feed state; must restore baseline after scenario |
| /resilience/* → Cache layer | Cached results must be keyed securely to prevent cross-user leakage (if multi-tenant in future) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | cache_key generation | mitigate | Use hash(distributor_id, sorted(bom_component_ids), scenario_type) to prevent collisions; do NOT allow user to specify cache_key directly |
| T-06-02 | Denial of Service | POST /resilience/* endpoints | mitigate | Validate bom_component_ids.length <= 200 (same limit as /graph/simulate); reject requests over this size |
| T-06-03 | Information Disclosure | live feed overrides | mitigate | After scenario computation, restore live feed state to baseline; do NOT leak overridden values in response |
| T-06-04 | Elevation of Privilege | scenario results | accept | Results are aggregate deltas (cost %, ETA days, risk scores) — no user data, pricing, or supplier secrets exposed. Public API (no auth required) |
| T-06-05 | Repudiation | scenario cache | accept | Cache is 1h TTL; no audit trail required for interview scenario exploration |
| T-06-06 | Denial of Service | cache table bloat | mitigate | Implement cache cleanup: expired records deleted hourly; add index on expires_at for efficient purge queries |

</threat_model>
