# Phase 3 — UI Design Contract

> Visual and interaction contract for the Live Feeds status widget added to the existing Dashboard page. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope reminder:** Phase 3 is primarily backend (LiveDataCache, APScheduler, 4 fetchers, CP-SAT/ETA injection). The only frontend deliverable is a compact `FeedStatusCard` section appended to `frontend/src/pages/Dashboard.tsx`, plus the `fetchFeedStatus()` API call in `frontend/src/services/api.ts`. This contract therefore extends the existing Dashboard design language rather than introducing new tokens.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (manual Tailwind utility composition — matches existing Dashboard convention) |
| Preset | not applicable |
| Component library | none (Tailwind v4.2 + framer-motion for animation; matches NavBar.tsx and Dashboard.tsx) |
| Icon library | lucide-react v1.7.0 (already in package.json) — use for status icons (`Activity`, `AlertTriangle`, `WifiOff`); plain Tailwind dot spans acceptable as fallback to match existing `w-1.5 h-1.5 rounded-full` pattern in Dashboard.tsx line 203 |
| Font | System stack: `-apple-system, BlinkMacSystemFont, 'Segoe UI', 'Roboto', 'Oxygen', 'Ubuntu', 'Cantarell', 'Fira Sans', 'Droid Sans', 'Helvetica Neue', sans-serif` (declared in `frontend/src/index.css` line 9–11) |

**Source rationale:** No `components.json` exists. Existing Dashboard.tsx (450 lines) is a mature, internally-consistent implementation using Tailwind utility classes + framer-motion + recharts. Introducing shadcn for a 4-row status widget would create design inconsistency across the 8 existing pages. The widget MUST adopt existing Dashboard tokens.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage in this widget |
|-------|-------|----------------------|
| xs | 4px (`gap-1`, `p-1`) | Status dot to label gap, micro padding |
| sm | 8px (`gap-2`, `p-2`, `py-2`) | Row vertical padding, badge inner padding |
| md | 16px (`gap-4`, `p-4`, `mb-4`) | Inter-row spacing inside the widget card not used (rows are dense at py-2); used for card outer padding consistency with existing KPI cards |
| lg | 20px (`p-5`) | Card inner padding — matches existing chart cards in Dashboard.tsx (line 225, 278, 314, 341, 365) |
| xl | 24px (`mb-6`) | Vertical gap between widget section and adjacent Dashboard sections — matches `mb-6` rhythm used between Dashboard rows (lines 215, 333, 402, 425) |

Exceptions: none. The widget reuses the existing Dashboard 4-step rhythm (4 / 8 / 16 / 20 / 24). Status dot is `w-1.5 h-1.5` (6px) — this is a decorative inline marker, not a layout token, and matches the existing "Real Data" pulse dot on Dashboard.tsx line 203.

**Line heights:** Tailwind default (`leading-normal` ~1.5) for body, `leading-tight` (1.25) for the card title. Both inherited from existing Dashboard convention.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Card title | `Live Feeds` |
| Card subtitle | `External signals refreshed every 15 minutes` |
| Row 1 feed name | `GPR Index` (NOT "Geopolitical Risk Index" — terse to match existing KPI label conventions) |
| Row 2 feed name | `ACLED Conflict` |
| Row 3 feed name | `IMF PortWatch` |
| Row 4 feed name | `FRED Freight` |
| Status badge: Live | `Live` (green dot + word) |
| Status badge: Stale | `Stale` (amber dot + word) |
| Status badge: Unavailable | `Unavailable` (slate dot + word) — NOT "Error", NOT "Down", NOT "Failed" |
| Timestamp format | Relative when < 1h: `2m ago`, `45m ago`. Absolute when ≥ 1h: `2026-04-16 14:32 UTC`. Use `Intl.DateTimeFormat` with `timeZone: 'UTC'` for the absolute form to keep deterministic display across user timezones |
| Empty state (initial load before first fetch) | Show all 4 rows with `—` in the timestamp column and the `Unavailable` badge. No separate empty-state copy needed — the rows themselves are the empty state |
| Error state (`/feeds/status` endpoint returns 500 or 401) | Single muted line replacing the 4 rows: `Feed status unavailable. Refresh to retry.` (no error icon — consistent with Dashboard.tsx line 104 silent-fail pattern) |
| Hover tooltip on each badge | `Last fetched: {ISO8601 timestamp}` — uses native `title` attribute, no custom tooltip component |
| Primary CTA | **None.** This widget is read-only by design. Phase 3 explicitly defers a manual `POST /admin/refresh-feeds` endpoint to v2 (per CONTEXT.md "Out of Scope" section) |
| Destructive actions | **None.** Status display only — no user mutations on this surface |

**Copywriting rules followed:**
- Each feed name is a recognizable acronym/product (matches the technical-audience tone of the rest of the app — "MPN", "Octopart", "VRP" all appear unexplained in existing UI)
- Status words are single tokens (Live / Stale / Unavailable) — parallel grammar
- No emojis in feed names or status (the app uses emojis for nav icons but never for data labels — see Dashboard.tsx KPI cards on lines 211–214)
- No exclamation marks, no marketing voice — matches the calm, instrumented tone of "from HuggingFace dataset" (line 211)

## Component Inventory (this phase)

| Component | File | Purpose | Dependencies |
|-----------|------|---------|--------------|
| `FeedStatusCard` | `frontend/src/pages/Dashboard.tsx` (inline, NOT a separate file unless it grows beyond ~80 lines) | Renders the 4-row Live Feeds widget | `framer-motion` (for fade-in `motion.div` matching the existing card animation pattern), `useEffect`/`useState` for polling, `feedsAPI.getStatus()` from api.ts |
| `feedsAPI` | `frontend/src/services/api.ts` | New exported object with `getStatus()` calling `GET /api/v1/feeds/status` | existing `api` axios instance |

**Polling strategy:** The widget polls `GET /feeds/status` every 60 seconds via `setInterval` inside `useEffect` (cleanup on unmount). The backend cache TTL is 15 min so 60s polling is well within freshness budget and avoids over-fetching. No SWR / React Query — matches existing `useEffect + axios` pattern (Dashboard.tsx lines 98–110).

**Layout placement:** Insert as a new row between the existing "Risk Radar by Category" row (Dashboard.tsx line 336) and "Navigation Cards" row (line 405). Use `grid grid-cols-5 gap-4 mb-6` wrapper with the FeedStatusCard occupying `col-span-5` (full width) — or `col-span-2` paired with a future expansion card filling `col-span-3`. **Phase 3 ships with `col-span-5` (full-width) for clarity and to leave horizontal room for the timestamp column.**

## Interaction States

| State | Visual treatment |
|-------|------------------|
| Default (Live) | Solid green dot (no animation) + "Live" text + relative or absolute timestamp |
| Default (Live, just fetched within last 60s) | Green dot with `animate-pulse` for 5 seconds after detected fetch update, then static. Optional polish — drop if it complicates implementation |
| Default (Stale) | Solid amber dot + "Stale" text + absolute timestamp showing how out-of-date the data is |
| Default (Unavailable) | Hollow slate dot (`border border-slate-500 bg-transparent`) + "Unavailable" text + `—` in timestamp column |
| Loading (initial mount, before first response) | All 4 rows show animated skeleton placeholder: `bg-slate-700/40 animate-pulse` rectangles at the timestamp and badge positions. Feed names render immediately (they are static) |
| Hover on row | `hover:bg-slate-900/50` row background (matches existing risky-component row pattern Dashboard.tsx line 384). Native `title` attribute reveals full ISO8601 fetched_at on the badge |
| Endpoint error | Single muted text line replaces row content (see Error state copy above). No retry button in Phase 3 |

## Accessibility

| Concern | Treatment |
|---------|-----------|
| Status indicated by color alone | NO — every status has a text label adjacent to the colored dot (`Live` / `Stale` / `Unavailable`). Color-blind users see the word |
| Color contrast (text on slate-800/60 card) | text-slate-200 on slate-900-equivalent background = ~12:1 (passes WCAG AAA). text-slate-400 = ~6:1 (passes AA). text-slate-500 used for subtitle only = ~4.6:1 (passes AA for non-essential text) |
| Pulse animation respect for reduced motion | Wrap `animate-pulse` usage in `motion-safe:animate-pulse` Tailwind variant so users with `prefers-reduced-motion` see no animation |
| Live region for status updates | Add `aria-live="polite"` to the widget container so screen readers announce status changes when polling updates the badges |
| Keyboard focus | Widget is read-only (no interactive elements in Phase 3). No focus management needed beyond default |
