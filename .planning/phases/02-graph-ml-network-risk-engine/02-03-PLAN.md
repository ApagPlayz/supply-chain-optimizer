**Monte Carlo algorithm (from CONTEXT.md Decision 3):**

Each of N=1,000 scenarios:
1. Sample distributor failures: for each distributor, fail it with probability = normalized_betweenness * STRESS_FACTOR (default 1.0)
2. A component is unfulfillable if ALL its supplying distributors failed (or it has no suppliers)
3. fulfillment_rate = n_fulfillable / n_bom
4. cost_inflation = 1.0 + (n_unfulfillable / n_bom) * 0.15

Output:
- P10 = 10th percentile of fulfillment_rates (worst scenarios)
- P50 = median fulfillment rate
- P90 = 90th percentile (best scenarios)
- EVaR_95 = mean cost_inflation of the worst-5% scenarios (by fulfillment rate)

N_SCENARIOS = 1000 is a module constant. The API endpoint always passes this constant — no user input controls iteration count (T-02-03 threat mitigation).

Module docstring:
  Monte Carlo cascade failure simulation.
  Runs N=1,000 SIR-style scenarios over the bipartite supply graph.
  Each scenario probabilistically fails distributors weighted by normalized betweenness
  centrality, then checks which BOM components become unfulfillable.
  Fixed seed=42 ensures reproducible output.
  N is a module constant — never controlled by user input.

Constants:
  N_SCENARIOS = 1000  (Fixed — not user-configurable per T-02-03)
  DEFAULT_SEED = 42   (Reproducibility)
  STRESS_FACTOR = 1.0 (Phase 3 will inject live stress; default = full betweenness)
  EMERGENCY_COST_PREMIUM = 0.15  (15% cost inflation per unfulfillable component)

SimulationResult dataclass fields: p10, p50, p90, evar_95, n_scenarios, seed

Helper _get_comp_to_dists(gs, bom_component_ids):
  For each BOM component cid, find all distributor IDs that have an edge to c_{cid}
  in the DiGraph (predecessors of the component node). Returns Dict[int, Set[int]].

run_monte_carlo(gs, bom_component_ids, n_scenarios=N_SCENARIOS, seed=DEFAULT_SEED):
  1. If bom_component_ids is empty, return SimulationResult(p10=1.0, p50=1.0, p90=1.0, evar_95=1.0, ...)
  2. Build comp_to_dists mapping using _get_comp_to_dists
  3. Build failure_probs dict: {did: min(betweenness[did] * STRESS_FACTOR, 1.0)}
  4. Initialize rng = random.Random(seed)
  5. For each of n_scenarios:
     a. For each distributor: add to failed_dists if rng.random() < failure_prob[did]
     b. For each BOM component: unfulfillable if supplying_dists is empty OR is a subset of failed_dists
     c. fulfillment_rate = (n_bom - n_unfulfillable) / n_bom
     d. inflation = 1.0 + (n_unfulfillable / n_bom) * EMERGENCY_COST_PREMIUM
     e. Append fulfillment_rate and inflation to respective lists
  6. Sort fulfillment_rates ascending
  7. P10 = fulfillment_rates[int(0.10 * n_scenarios)] (clamped to valid index)
     P50 = fulfillment_rates[int(0.50 * n_scenarios)]
     P90 = fulfillment_rates[int(0.90 * n_scenarios)]
  8. EVaR: pair fulfillment_rates with cost_inflations, sort by rate ascending,
     take first 5% entries, evar_95 = mean of their cost_inflations
  9. Return SimulationResult(p10, p50, p90, evar_95, n_scenarios, seed)
