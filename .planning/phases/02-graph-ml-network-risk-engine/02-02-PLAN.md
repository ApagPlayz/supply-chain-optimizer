Expected metric properties from fixture:
- Betweenness: dist 1 (DigiKey) carries ALL 10 components → highest betweenness
  dist 2 carries only comps 1-5 → lower betweenness
  dist 3 carries nothing (no offers in training set after holdout) → betweenness=0.0 normalized
- PageRank: dist 1 should have highest PageRank (most edges)
- k_core: components 6-10 have degree=1 in undirected graph (only dist 1) → core_number=1
- single_source_component_ids: components 6-10 have only dist 1 as stocked distributor
  → single_source_component_ids should include {6,7,8,9,10}
- hhi_by_category:
    Microcontrollers (comps 1-5): dist 1 stock=500, dist 2 stock=500 → shares 50%/50% → HHI=5000
    Op-Amps (comps 6-10): dist 1 stock=250 → share=100% → HHI=10000
- fiedler: fixture graph has 2 main connected components (dist 3 is isolated) → 0.0 for full
  graph but non-zero for largest connected component (builder uses LCC)
